# Specs

Specifications for KoroMind services, features, and issues.

**Read [SPEC_FORMAT.md](./SPEC_FORMAT.md) before creating or editing specs.**

## Quick Reference

- Specs describe **services/usecases**, not files
- One spec per mental model ("how does X work?")
- Max 50 lines, frontmatter required
- Naming: `service-*.md`, `feature-*.md`, `bug-*.md`, `security-*.md`
